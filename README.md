## Project Context

The growing demand for fast and accessible online services, combined with the continuous evolution of digital technologies, has strongly contributed to the expansion of e-stores over recent years. This rapid growth has intensified competition, both in terms of technological innovation and commercial offerings.

Despite this evolution, many existing e-commerce platforms still present significant shortcomings, particularly regarding system reliability, usability, ergonomics, and overall user experience.

This project is part of this context. Its objective is to define, analyze, and structure the requirements of an online e-store dedicated to the sale of sports equipment, with a specific focus on technical, functional, and organizational aspects necessary for the design of an efficient, secure, and reliable system.

## Project Objectives

The main goals of this project are:

To identify and formalize the requirements of a sports equipment e-store

To structure these requirements following a requirements engineering approach

To distinguish clearly between:

the Environment Book (context and constraints)

the System Book (system behavior and structure)

To automate the generation of a final requirements document in PDF and HTML formats

## Repository Organization & Workflow

To ensure a clean and collaborative development process, we adopted a branch-based workflow:

main
The main branch containing stable and validated versions of the project.

feat2/branche-requirement
Dedicated to the writing and refinement of the requirements documents (Environment Book and System Book).

Documentation branch
Used for updates and improvements to the README.md file and project documentation.

This workflow allows:

clear separation of responsibilities

better version control

easier review and validation of changes

## Technologies & Tools
